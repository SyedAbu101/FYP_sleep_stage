## Hardware Platform (Deployment)

### Seeed XIAO nRF52840 Sense

| Specification | Value |
|---------------|-------|
| MCU | Nordic nRF52840, ARM Cortex-M4F @ 64 MHz |
| FPU | Hardware single-precision FPU (no fixed-point needed) |
| Flash | 1 MB |
| RAM | 256 KB |
| IMU | LSM6DS3 — 6-axis (3-axis accel + 3-axis gyro), I2C |
| Microphone | MP34DT06JTR PDM @ 16 kHz |
| RGB LED | Active-LOW (LEDR / LEDG / LEDB) |
| Connectivity | Bluetooth 5.0, USB-C |
| Voltage | 3.3 V |
| Form factor | 20 mm × 17.5 mm |

### Firmware Libraries

| Library | Purpose |
|---------|---------|
| `PDM.h` (Seeed board package) | PDM microphone driver |
| `Wire.h` | I2C for IMU |
| `Eloquent::ML::Port::XGBClassifier` | Inference (generated by micromlgen) |

### Firmware Architecture

- IMU sampled at 1 Hz; statistics accumulated over 30-second epochs
- PDM microphone: 256-sample windows streamed through a radix-2 DIT FFT (float32, Cortex-M4F FPU); running sums and sum-of-squares accumulated — no 30-second waveform buffer
- Patient baseline updated online each epoch: `p_acc_mean = pat.acc_sum / pat.n_epochs` (O(1), converges continuously)
- Thread safety: `noInterrupts()` / `interrupts()` around ISR buffer copy
- RGB LED output updated after each epoch: **Red = Wake, Green = NREM, Blue = REM**
- Total audio-related RAM: ~4.8 KB (dominated by `fft_out[512]` complex float buffer, 2 KB)

## Model Specifications

### Final Model: XGBoost

```python
XGBClassifier(
    n_estimators=20,           # Reduced from 100 to fit nRF52840 1 MB flash
    max_depth=6,
    learning_rate=0.1,
    subsample=0.8,
    colsample_bytree=0.8,
    objective='multi:softmax',
    num_class=3,
    random_state=42
)
# Class imbalance: compute_class_weight('balanced', classes, y_train)
# applied as sample_weight in model.fit() — recomputed per fold
```

**Why n_estimators=20:** The full 100-tree model exceeds the 1 MB flash budget when exported to a C header via `micromlgen`. 20 trees fits within budget while achieving comparable Kappa (0.3234 vs full-model values).

**Export:** `micromlgen.port(model)` generates `Eloquent::ML::Port::XGBClassifier` — a self-contained C++ class with no external dependencies.

## Experimental Setup

### Cross-Validation
- **Method:** `GroupKFold(n_splits=5)` — patient as group
- **Guarantees:** Zero patient overlap between train and test splits
- **Scaler:** `StandardScaler` fit on training split only, applied to test split

### Class Imbalance
- **Method:** `sklearn.utils.class_weight.compute_class_weight('balanced', classes, y_train)` per fold
- Applied as `sample_weight` in `model.fit()` — not `scale_pos_weight` (which is for binary tasks)
- Recomputed each fold so weights reflect the actual training distribution

### Reproducibility
- `random_state=42` in XGBClassifier
- `np.random.seed(42)` at script start
- Feature selection uses `results/audio_feature_importance.csv` `is_audio` column (not keyword matching) — deterministic

## Key Findings

1. **Patient-specific normalisation** is the single most impactful feature group (~21% importance). The online running-mean estimator is strictly superior to a startup-frozen baseline because it converges toward the true individual mean as the session progresses.

2. **Audio features contribute positively.** Including the top-20 audio features (12% importance) in the 118-feature model achieves Kappa = 0.3234 with 20 trees. The microphone provides useful spectral and energy cues despite the device's pillow placement.

3. **REM remains the hardest class** (F1 = 0.364, recall = 40.6%). REM movement patterns overlap significantly with light NREM on IMU-only sensing — most confusion is NREM↔REM.

4. **Embedded deployment is feasible.** The 20-tree XGBoost model fits within the nRF52840's 1 MB flash, runs inference in ~12 ms, and uses ~4.8 KB audio RAM — well within the 256 KB RAM budget.
